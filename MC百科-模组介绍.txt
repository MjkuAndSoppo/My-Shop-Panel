【模组简介】
================================================================================

  My Shop Panel 是一款轻量级 Minecraft 经济模组。

  它为服务器提供了完整的玩家交易生态：独立的 MSPP（MS.P.Points）点数货币、
  玩家自由挂单市场、以及 OP 可配置的官方世界商店。所有功能统一通过一枚「报价
  终端」物品操作，不依赖任何原版容器界面。

================================================================================
【核心特性】
================================================================================

  1. 独立 GUI 渲染
     所有界面继承 Screen 自绘，完全脱离原版箱子/工作台等容器 GUI，与任何
     其他模组的界面系统零冲突。

  2. MSPP 点数系统
     独立于原版经验的虚拟货币。通过 WorldSavedData 持久化存储，支持负数余
     额（即允许玩家透支/欠款）。管理员可自由增发、回收、调整。

  3. 玩家自由市场
     玩家自主上架背包物品、浏览所有挂单、购买他人商品、撤回自己的挂单，
     形成完整的 C2C 交易循环。价格完全由玩家自由设定。

  4. 世界商店（OP 商店）
     管理员可配置 Buy（收购）和 Sell（出售）两种模式的官方商店条目。库存
     可设为有限或无限模式，支持实时热编辑。

  5. 三步提交事务
     Prepare（生成预览）→ Confirm（用户确认）→ Commit（落盘执行）。
     若中途取消、超时、断线，事务自动回滚。涉及透支操作会高亮警告。

  6. 冗余仓库
     当玩家离线时发生挂单下架，物品自动寄存至冗余仓库，玩家下次上线后
     可手动取回，杜绝离线丢物品的问题。

  7. 黑名单机制
     管理员可将指定物品（如报价终端本身）加入黑名单，禁止其在市场上架。

  8. I 键快捷入口
     背包中存有报价终端时，按 I 键即可快速打开主菜单，无需手持使用。

【模组联动】
================================================================================

  —— ProjectE ——
  安装 ProjectE 后，管理员在世界商店中新增商品时，默认标价会自动填入该物品
  的 EMC 值。通过反射调用，不硬依赖 ProjectE，未安装时默认标价为 1 MSPP。

【指令】
================================================================================

  —— /msp —— 点数管理 ——
  /msp get  <玩家>             查询玩家余额
  /msp set  <玩家> <金额>      设置余额（可为负）
  /msp add  <玩家> <金额>      增加余额
  /msp cut  <玩家> <金额>      扣减余额
  /msp pay  <玩家> <金额>      转账给其他玩家（无需OP）

  —— /MSPB —— 报价管理（需OP）——
  /MSPB up <单价> [数量]       以服务器名义上架手中物品
  /MSPB dn <展示ID>            下架指定报价单

  —— /MSPEdit —— 编辑模式（需OP）——
  /MSPEdit true                开启编辑模式，主菜单出现编辑按钮
  /MSPEdit false               关闭编辑模式

  —— /MSPBlacklist —— 黑名单（需OP）——
  /MSPBlacklist add <注册名>    添加黑名单物品
  /MSPBlacklist remove <注册名> 移除黑名单物品
  /MSPBlacklist list            查看黑名单列表

【操作流程】
================================================================================

  ─ 上架物品 ──────────────────────────────────────
  右键终端 → 玩家市场 → 「上架物品」
  → 9×4 背包网格中点击物品 → 输入标价和数量 → 确认
  （物品从背包扣除，挂单创建）

  ─ 购买物品 ──────────────────────────────────────
  右键终端 → 玩家市场 → 浏览全部挂单
  → 点击他人挂单 → 确认弹窗 → 扣款 → 获得物品
  （透支时弹窗高亮警告）

  ─ 撤回挂单 ──────────────────────────────────────
  玩家市场 → 切换「我的挂单」→ 点击撤回
  （物品退回背包 / 背包满则掉落地面）

  ─ 世界商店（玩家）──────────────────────────────
  右键终端 → 主菜单 → 世界商店
  SELLING 条目（购买）：选择数量 → 扣款 → 获得物品
  BUYING  条目（出售）：选择数量 → 扣除背包物品 → 获得MSPP
  右键 BUYING 条目可 1.3倍 价格买回

  ─ 世界商店管理 ─────────────────────────────────
  潜行右键终端（需OP）→ OP管理
  或 /MSPEdit true → 主菜单出现编辑按钮
  可：新增/删除商品、调整库存、切换买卖模式、切换库存类型

  ─ 冗余仓库取回 ─────────────────────────────────
  上线后收到提示 → 右键终端 → 冗余仓库
  → 多选物品 → 「取回」或「一键取回」

【FAQ】
================================================================================

  Q：客户端需要安装吗？
  A：需要。所有 GUI 由客户端渲染，服务端和客户端须同时安装。

  Q：MSPP 和原版经验是什么关系？
  A：完全独立。MSPP 通过 WorldSavedData 存储，与原版经验系统无任何关联。

  Q：玩家余额可以为负数？
  A：可以。模组不设余额下限，允许透支。管理员可随时通过 /msp set 调整。

  Q：物品会丢失吗？
  A：不会。背包满时物品掉落地面（player.drop()）。离线玩家下架的物品会进
     入冗余仓库，上线后可取回。

  Q：和其他容器类 UI 模组冲突吗？
  A：不冲突。所有界面继承 Screen 而非 AbstractContainerScreen，完全脱离
     原版容器体系。

  Q：生存模式如何获得报价终端？
  A：用工作台 + 熔炉 + 玻璃 + 煤炭块无序合成。或由管理员通过 /give 指令给予。
